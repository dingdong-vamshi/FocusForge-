# Entity Relationship Diagram

The ER Diagram represents the database structure of the FocusForge system.
It models how data is stored and related in MongoDB collections.

The main entity is **User**, which is connected to ActivitySession, Goal, Reminder, FocusSession, and Report.
A user can generate multiple activity sessions, set multiple goals, and receive reminders.

The **ActivitySession** entity stores browsing or usage records of the user.
The **Report** entity is a derived entity generated by aggregating multiple activity sessions for a particular day and calculating productivity metrics.

All relationships in the system are primarily one-to-many, where one user owns multiple records.
This design ensures proper data organization, efficient querying, and accurate analytics generation.
